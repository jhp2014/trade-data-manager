# ADR-015: `-pl` 플래그 → 가격 라인

**상태**: Accepted  
**날짜**: 2026-05-10

## 맥락

입력 파싱에서 가격 라인 데이터를 일반 옵션과 분리하지 않으면 차트 컴포넌트가 받는 타입이 오염된다.
가격은 숫자이고 옵션은 문자열 값이라 처리 방식도 다르다.

## 결정

- 입력 형식에서 `-pl` 플래그로 가격 라인을 지정한다 (예: `-pl 51000|41000`).
- 값은 `"|"` 구분 다중 가격.
- `ChartModalTarget.priceLines: Record<string, number[]>` 필드로 전달된다.
  - key는 컬럼 label (예: `"TARGET"`).
  - value는 파싱된 숫자 배열.

## 차트 적용 규칙

- **일봉**: 가격 그대로 수평선 표시. Y축 레이블에 가격 표시.
- **분봉**: `prevCloseKrx` 또는 `prevCloseNxt` 기준으로 % 변환 후 표시.
  - `prevClose`가 null이면 미표시.
- **오버레이**: 미적용 (% 단위 종목별 시리즈가 다수라 의미 약함 — 추후 요구 시 별도 결정).
- 차트 레이블: `line_` prefix가 있으면 제거 후 표시 (`line_target` → `target`).

## 결과

- 일봉/분봉 차트에 수평선이 그려짐.
- 파싱 불가 토큰은 제외되어 투명하게 처리됨.
