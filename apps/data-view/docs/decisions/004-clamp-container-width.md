# ADR-004: 컨테이너 너비 `clamp()` 적용

## 상태

Accepted (2026-05-08)

## 맥락

앱 메인 페이지 컨테이너의 `max-width`가 `1400px → 1760px → 2100px`으로 미디어 쿼리 두 개를 통해 단계적으로 확장됐다. 임계 해상도(1800px, 2200px) 직전에서 컨테이너 폭이 갑자기 점프하는 시각적 불연속성이 발생했다. QHD 이상 모니터에서 작업할 때 이 점프가 두드러졌다.

## 검토한 대안

- **A: media query 유지** — 현재 구조 그대로. 기각: 경계에서의 점프가 사용자 경험을 해치며, 임계값을 조정할 때마다 두 미디어 쿼리를 함께 관리해야 한다.
- **B: `clamp(1200px, 78vw, 2100px)` (채택)** — 단 한 줄로 min~max 범위에서 뷰포트 비례로 부드럽게 변화. FHD(1920px) → ~1498px, QHD(2560px) → ~1996px, 4K(3840px) → 2100px(max 클램프).
- **C: JS 기반(ResizeObserver + inline style)** — 자바스크립트로 동적 계산. 기각: CSS만으로 해결 가능한 문제에 런타임 의존성을 추가할 필요가 없다.

## 결정

**B안** 채택. `max-width: clamp(1200px, 78vw, 2100px)` 한 줄로 미디어 쿼리 두 개를 대체했다. 78vw 비율은 시각 검수 후 75~82vw 범위에서 조정 가능하도록 파일 상단 주석에 명시했다.

## 결과

- **장점**: 브라우저 폭을 1200px~4000px까지 연속으로 변경해도 점프 없이 부드럽게 변한다. 코드 줄 수 감소(미디어 쿼리 블록 2개 → 1줄).
- **단점/한계**: 뷰포트 1280px 이하에서는 `1200px` min에 고정되어 좌우 여백이 거의 없다. 1280px 미만 대응은 별도 정책(패딩 축소 등) 필요.

## 관련

- 코드: `src/app/(main)/layout.module.css`
- 후속 ADR: 없음
